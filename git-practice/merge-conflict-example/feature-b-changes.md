# feature-b 브랜치 — 전문적 톤 (강사 사전 참고용)

> 이 파일은 **강사가 라이브 전 시나리오 이해용**으로 보는 자료입니다.
> 라이브에선 학생이 직접 `회사_소개.md` 파일을 아래 내용으로 수정합니다.

---

## 갈래: `feature-b`

## 수정할 파일: `회사_소개.md`

## 수정 후 내용 (딱 한 줄)

```
당사는 데이터 기반의 AI 교육 솔루션을 제공하는 전문 기업입니다.
```

---

## 의도

- 전문적·기업 소개 톤. 회사 홈페이지 About 페이지 본문 톤.
- "당사" 격식체, "데이터 기반", "솔루션", "전문 기업" — 보고서 어휘.
- main의 원본 한 줄 ("저희는 AI 교육 회사입니다.") 과 **같은 자리·같은 줄**을 다르게 고침.
- feature-a와 **명확히 다른 단어**로 채워야 충돌 마커가 깨끗하게 떨어짐.

---

## 강사 멘트 (라이브용)

> "이 갈래는 전문적 톤이에요. 회사 홈페이지 About 페이지 본문 같은 거. main으로 한 번 돌아가서, 새 갈래 만들고, 같은 파일 같은 줄을 다르게 고쳐볼게요."

```bash
git switch main
git switch -c feature-b
# 회사_소개.md 위 내용으로 수정 + 저장
git add 회사_소개.md
git commit -m "전문적 톤으로 수정 (feature-b)"
```

---

## 충돌 발생 조건 (확인용)

이 두 갈래(feature-a, feature-b)는:
- **같은 파일** (`회사_소개.md`)
- **같은 줄(1번 줄)**
- **다른 내용**

→ main이 둘 다 흡수하려 하면 두 번째 merge에서 무조건 `CONFLICT`.

(만약 충돌 안 뜨면: 한쪽 갈래가 main 기준이 아닌 다른 갈래에서 분기됐을 가능성. `git switch main` 으로 돌아간 뒤 새 갈래 만드는지 확인.)
