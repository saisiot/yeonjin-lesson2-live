# feature-a 브랜치 — 친근한 톤 (강사 사전 참고용)

> 이 파일은 **강사가 라이브 전 시나리오 이해용**으로 보는 자료입니다.
> 라이브에선 학생이 직접 `회사_소개.md` 파일을 아래 내용으로 수정합니다.

---

## 갈래: `feature-a`

## 수정할 파일: `회사_소개.md`

## 수정 후 내용 (딱 한 줄)

```
안녕하세요! 저희는 AI 교육을 함께 만들어가는 따뜻한 회사예요.
```

---

## 의도

- 친근한 마케팅 톤. 인스타·블로그 본문 톤.
- "저희" 1인칭, 느낌표, 정서적 형용사("따뜻한").
- main의 원본 한 줄 ("저희는 AI 교육 회사입니다.") 과 **같은 자리·같은 줄**을 다르게 고침 — 충돌 만들기 위한 핵심 조건.

---

## 강사 멘트 (라이브용)

> "이 갈래는 친근한 톤이에요. 인스타에 올릴 회사 소개 같은 거. 한 줄로 바꿔 저장하고 commit 해볼게요."

```bash
git switch -c feature-a
# 회사_소개.md 위 내용으로 수정 + 저장
git add 회사_소개.md
git commit -m "친근한 톤으로 수정 (feature-a)"
```
