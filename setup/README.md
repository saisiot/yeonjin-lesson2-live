# 2회차 사전 설치 안내

연진 님, 2회차는 **손으로 만져보는 시간**입니다. 1회차에서 "왜 이 도구들이 생겼는지"는 봤으니, 이번엔 그 도구들을 노트북에 직접 올려놓고 한 줄씩 굴려봅니다. 그래서 라이브 전에 도구 몇 가지를 미리 깔아두실 부탁이 있어요. 설치 자체가 그날 강의 시간을 잡아먹지 않도록, 라이브에선 "잘 깔렸는지 점검"만 하고 바로 손으로 들어갑니다.

---

## 이번 회차에 필요한 4가지

| # | 이름 | 1회차에서 들었던 한 줄 | 이번에 하는 일 |
|---|---|---|---|
| 1 | **VSCode** | 글 쓰는 종이의 진화 | 코드 입력하고 실행하는 작업창 |
| 2 | **Python 3.11+** | 실행 환경 | VSCode에서 쓴 글을 실제로 돌리는 통역기 |
| 3 | **Git** | 이력 도구 | 글 고친 흔적을 매듭으로 묶어두는 장치 |
| 4 | **GitHub 계정** | (1회차엔 등장 X) | 위에서 묶은 매듭을 클라우드에 올려두는 자리 |

> 1~3번은 본인 노트북에 설치하는 프로그램이고, 4번은 웹사이트에서 계정만 만드시면 됩니다.

---

## 설치 순서

순서대로 진행하시는 걸 권합니다. 뒤 단계가 앞 단계를 전제로 하기 때문입니다.

1. **VSCode** → [vscode_install.md](./vscode_install.md)
2. **Python** → [python_install.md](./python_install.md)
3. **Git** → [git_install.md](./git_install.md)
4. **GitHub 계정** → 웹에서 https://github.com/ 들어가서 이메일 인증까지. 별도 가이드는 따로 없습니다.

각 가이드는 Mac과 Windows 둘 다 다룹니다. 본인 노트북에 맞는 쪽만 따라가시면 됩니다.

---

## 라이브 전 자가 점검 체크리스트

설치를 마치셨으면 터미널(Mac) 또는 PowerShell·Git Bash(Windows)를 열고 아래 한 줄씩 쳐보세요. 버전 숫자가 어떻게 나오든 **에러 없이 숫자가 보이면 통과**입니다.

```
code --version
```
> VSCode가 명령줄에 잡히는지 확인. (Mac에서 안 잡히면 정상이에요. VSCode가 그냥 켜지기만 해도 OK)

```
python --version
```
> Mac에서 위 명령으로 안 잡히면 `python3 --version`으로 시도하시면 됩니다.

```
git --version
```
> 숫자가 보이면 통과.

```
git config --global user.name
git config --global user.email
```
> 본인 이름과 이메일이 보이면 통과. 안 나오면 git_install.md의 4단계로 돌아가세요.

GitHub 계정은 https://github.com/ 로그인이 되면 통과입니다.

- 설치가 끝까지 안 되어도 라이브엔 오세요. 강사 화면 보면서 진행할 수 있고, 짬짬이 같이 깔아도 됩니다.
